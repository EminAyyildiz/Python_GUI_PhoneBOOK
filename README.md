# Python_GUI_PhoneBOOK


# Bu projedeki temel amacım python class, fonksiyon, ve GUI yöntemlerini pekiştirmektir. Bu proejde bir arayüz oluşturabilmek için python tkinter kütüphanesinden yararlandım. Bu projedeki temel hedef kullanıcının sisteme bireyeleri iş ve ev telefon numaraları ile kaydedip bunları listeleyebileceği bir telefon defteri tasarlamaktır. Bu telefon defteri kullanıcıdan adı-soyadı-iş telefonu-ev telefonu- olmak üzere 4 input ifadesi almaktadır. Kullanıcı bu ifadelerden herhangi birini girmediği sürece sistem arka plandaki txt dosyasına kaydetmeye izin vermemektedir. Kullanıcı tüm bilgileri girdiğinde ve Add contact butonuna bastığında ekranda sisteme başarıyla kaydedildiğine dair  bilgi mesajı çıkmaktadır. Ayrıca kullanıcı yine uygulama arayüzünde bulunan listeleme butonu ile kaydedilmiş tüm bilgileri ekranda gösterebilmektedir. Son olarak çıkış butonu sayesinde de uygulamadan çıkış yapabilmetktedir. Ancak unutulmaması gereken kullanıcı çıkış yapsa dahi bilgiler txt dosyasına anlık olarak kaydedilmekte ve veri kaybı olmamaktadır. Sisteme dair tüm görüntüler bu proje altında GitHub da bulunmaktadır. 


# My main purpose in this project is to consolidate python class, function, and GUI methods. I used the python tkinter library to create an interface in this project. The main goal in this project is to design a phone book where the user can register and list the individuals in the system with their work and home phone numbers. This phone book takes 4 input expressions from the user: name, surname, work phone, home phone. Unless the user enters any of these statements, the system does not allow saving to the txt file in the background. When the user enters all the information and presses the Add contact button, an information message appears on the screen that it has been successfully registered to the system. In addition, the user can display all the recorded information on the screen with the listing button on the application interface. Finally, it is possible to exit the application with the exit button. However, it should not be forgotten that even if the user logs out, the information is instantly saved in the txt file and there is no data loss. All images of the system are available on GitHub under this project.


# Mein Hauptzweck in diesem Projekt ist es, Python-Klassen, -Funktionen und -GUI-Methoden zu konsolidieren. Ich habe die Python-tkinter-Bibliothek verwendet, um eine Schnittstelle in diesem Projekt zu erstellen. Das Hauptziel dieses Projekts ist die Gestaltung eines Telefonbuchs, in dem der Benutzer die Personen im System mit ihren geschäftlichen und privaten Telefonnummern registrieren und auflisten kann. Dieses Telefonbuch nimmt 4 Eingabeausdrücke vom Benutzer entgegen: Vorname, Nachname, Telefon geschäftlich, Telefon privat. Wenn der Benutzer keine dieser Anweisungen eingibt, lässt das System das Speichern in der TXT-Datei im Hintergrund nicht zu. Wenn der Benutzer alle Informationen eingibt und auf die Schaltfläche Kontakt hinzufügen klickt, erscheint eine Informationsmeldung auf dem Bildschirm, dass er erfolgreich im System registriert wurde. Darüber hinaus kann der Benutzer alle aufgezeichneten Informationen auf dem Bildschirm mit der Listenschaltfläche auf der Anwendungsoberfläche anzeigen. Schließlich ist es möglich, die Anwendung mit der Exit-Schaltfläche zu beenden. Es sollte jedoch nicht vergessen werden, dass selbst wenn sich der Benutzer abmeldet, die Informationen sofort in der txt-Datei gespeichert werden und es keinen Datenverlust gibt. Alle Bilder des Systems sind auf GitHub unter diesem Projekt verfügbar.
